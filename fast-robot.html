Advantages and Disadvantages of Each Method

Each data transmission method has trade-offs. Sending datapoints live allows real-time monitoring, useful for debugging short operations. However, Bluetooth limitations cause data clipping, which is problematic for complex motions. The array method, in contrast, enables rapid data storage and transfer, processing data efficiently via the notification handler. This is ideal for post-analysis, such as optimizing a stunt, flip, or other fast manuever. The Artemis board has 384 kB of RAM (3,072,000 bits). At a 150 Hz sampling rate, 5 seconds of 16-bit values require 12,000 bits. This allows storing up to 256 such sets, assuming full memory availability. However, some of the memomry will be held by strings and the program so the capacity is less.

Effective Data Rate

To assess how message length impacts BLE data transfer rates, I developed a SEND_REPLIES command that accepts varies the message length from 5-150 bytes and tracked the time it took to complete each message. I then plotted this relationship and shows an inverse correlation between the transfer rate and message length. I'd imagine that fast short messages have more overhead and that fewer, longer message are prefereable in this case.

Effective Data Rate

Increasing the data transmission rate from the robot to the computer does not result in data loss. I verified this by sending an array of 1,000 timestamped entries from the Artemis to the computer, all of which were successfully received. However, since the data points were floats, the lower overhead may have contributed to efficient transmission.
